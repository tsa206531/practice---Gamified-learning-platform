# 本地一鍵啟動架構說明：Spring Boot + PostgreSQL + Next.js + Docker Compose（V411_17REV2）

本文整理自 L1/technology.promet 的技術選型與你的問題，目標是用前端工程師也能快速理解與落地的方式，解釋 Spring Boot、PostgreSQL、Docker Compose 是什麼、要怎麼做，以及會影響哪些開發流程。

## 這些是什麼

- Spring Boot
  - Java 的後端框架，用來建立 REST API。帶有快速啟動預設、內建嵌入式 Web 伺服器（通常是 Tomcat），很適合做 BFF/微服務。
  - 對前端同學來說，可以把它想成 Node.js + Express 的 Java 版本：型別安全強、資料庫整合成熟（JPA/Hibernate）、在生產環境很常見。

- PostgreSQL
  - 開源關聯式資料庫（SQL），在交易一致性、擴充功能、JSON 欄位支援等方面很強。是後端常用的主力資料庫之一。

- Docker Compose
  - 用一份 Compose 檔定義多個容器服務（前端、後端、資料庫），一個指令就能一起啟動/關閉，確保每個人的本地環境一致，減少「在我電腦可以」的問題。

## 要怎麼做（本地一鍵啟動的實作藍圖）

### 建議專案結構
- frontend/：你的 Next.js 專案（已存在）
- backend/：Spring Boot 專案（新增）
- docker-compose.yml：放在專案根目錄
- .env / .env.example：集中管理環境變數
- volumes/（可選）：資料庫持久化資料卷

### 流程概念
1) `docker-compose up` 會啟動三個服務：
   - frontend：Next.js（dev 或 build 後的產物）
   - backend：Spring Boot（提供 REST API，例如 /api/courses）
   - db：PostgreSQL（提供資料）
2) frontend 透過環境變數指向 backend（例如 http://backend:8080 或 http://localhost:8080）
3) backend 透過環境變數指向 db（postgres://...）

### docker-compose.yml 範例（dev 版）
```yaml
services:
  db:
    image: postgres:16
    container_name: postgres
    environment:
      POSTGRES_DB: appdb
      POSTGRES_USER: appuser
      POSTGRES_PASSWORD: apppass
    ports:
      - "5432:5432"
    volumes:
      - db_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U appuser -d appdb"]
      interval: 5s
      timeout: 5s
      retries: 5

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: backend
    environment:
      SPRING_PROFILES_ACTIVE: docker
      SPRING_DATASOURCE_URL: jdbc:postgresql://db:5432/appdb
      SPRING_DATASOURCE_USERNAME: appuser
      SPRING_DATASOURCE_PASSWORD: apppass
    depends_on:
      db:
        condition: service_healthy
    ports:
      - "8080:8080"

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: frontend
    environment:
      NEXT_PUBLIC_API_BASE_URL: http://localhost:8080
      # 若用容器間呼叫（或反向代理）也可設為 http://backend:8080
    ports:
      - "3000:3000"
    depends_on:
      - backend

volumes:
  db_data:
```

### backend/Dockerfile 範例（Spring Boot）
```dockerfile
# Build stage
FROM maven:3.9-eclipse-temurin-21 AS build
WORKDIR /app
COPY pom.xml .
RUN mvn -q -e -DskipTests dependency:go-offline
COPY src ./src
RUN mvn -q -DskipTests package

# Run stage
FROM eclipse-temurin:21-jre
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java","-jar","/app/app.jar"]
```

### frontend/Dockerfile 範例（Next.js）
```dockerfile
FROM node:20-alpine AS deps
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN corepack enable && pnpm i --frozen-lockfile

FROM node:20-alpine AS build
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN corepack enable && pnpm build

FROM node:20-alpine AS run
WORKDIR /app
ENV NODE_ENV=production
COPY --from=build /app/.next ./.next
COPY --from=build /app/package.json ./
COPY --from=deps /app/node_modules ./node_modules
EXPOSE 3000
CMD ["pnpm","start"]
```

## Spring Boot 最小骨架（給前端同學的地圖）
- Application 啟動類別
- Controller（定義 API 路徑）
- Model/Entity（對應 DB table）
- Repository（資料存取）
- application.yml（設定：DB 連線、CORS、port 等）
- Flyway（或 Liquibase）資料庫 migration

### application.yml（docker profile）
```yaml
server:
  port: 8080

spring:
  datasource:
    url: ${SPRING_DATASOURCE_URL}
    username: ${SPRING_DATASOURCE_USERNAME}
    password: ${SPRING_DATASOURCE_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: validate
    properties:
      hibernate:
        dialect: org.hibernate.dialect.PostgreSQLDialect
  flyway:
    enabled: true
    locations: classpath:db/migration

# 簡單 CORS 設定（或用全域 WebMvcConfigurer）
cors:
  allowed-origins: "http://localhost:3000"
  allowed-methods: "GET,POST,PUT,DELETE,OPTIONS"
```

### 簡單 Controller 範例（健康檢查 + 課程列表）
```java
@RestController
@RequestMapping("/api")
public class CourseController {

  @GetMapping("/health")
  public Map<String, String> health() {
    return Map.of("status", "ok");
  }

  @GetMapping("/courses")
  public List<CourseDto> list() {
    // 這裡先回 mock，等之後接 Repository
    return List.of(new CourseDto("intro-to-react", "Intro to React"));
  }
}
```

### DB migration（Flyway）
- `src/main/resources/db/migration/V1__init.sql`
```sql
CREATE TABLE courses (
  id SERIAL PRIMARY KEY,
  slug TEXT UNIQUE NOT NULL,
  title TEXT NOT NULL
);
```

## 前端怎麼接
- 在 Next.js 裡使用 `process.env.NEXT_PUBLIC_API_BASE_URL` 當 base URL
```ts
const base = process.env.NEXT_PUBLIC_API_BASE_URL || "http://localhost:8080";

export async function getCourses() {
  const res = await fetch(`${base}/api/courses`, { cache: "no-store" });
  if (!res.ok) throw new Error("Failed to fetch courses");
  return res.json();
}
```
- App Router 用法
```tsx
export default async function CoursesPage() {
  const courses = await getCourses();
  return <CourseList data={courses} />;
}
```

## 會影響什麼
- 開發流程
  - 不需要本地安裝 Postgres/Java，直接 `docker-compose up` 就能跑；第一次會較慢（下載映像、建置）。
  - 也可以選擇本機直跑前端 dev server（`pnpm dev`），backend+db 用 compose 啟動，前端照樣接 `http://localhost:8080`。
- 設定與環境變數
  - 需要 `.env`/`.env.local` 管理 API base URL、DB 密碼、JWT 秘鑰等，並在 compose 中傳入容器。
  - 建議建立 `.env.example` 範本。
- CORS 與網路
  - 前端在 `http://localhost:3000`，後端在 `http://localhost:8080`，要允許 CORS（或用反向代理/Next 中間層轉發）。
- 資料庫狀態
  - 使用 Docker volume 保存資料。若要重置資料，刪除 volume 即可。
- CI/CD
  - 之後可用同一份 compose 跑本地 E2E；上線時可改 K8s 或雲服務，但容器鏡像可重用。
- 版本與體積
  - 映像體積較大、建置較久。可用多階段建置、瘦身 JRE、node:alpine 等最佳化。
- 型別/契約
  - 建議用 OpenAPI 或共用型別（例如 Spring 產出 OpenAPI spec，前端用 openapi-typescript 產生型別）避免契約不一致。

## 最短可跑通的任務清單（建議落地順序）
1) 新增 backend 骨架（Spring Boot minimal REST + health）、Dockerfile、Flyway V1 migration。
2) 新增 docker-compose.yml：db + backend + frontend，並於 frontend 設定 `NEXT_PUBLIC_API_BASE_URL`。
3) 前端新增簡單 API 呼叫（/api/health 或 /api/courses），把頁面資料改由後端提供。
4) 新增 `.env.example`、`README`，說明如何 `docker-compose up`。

## 後續可逐步加強
- 認證（JWT/Session）與授權（RBAC/ABAC）
- 資料模型（Course, Challenge, User ...）與資料庫關聯
- 後端測試（JUnit）、觀測（Sentry/OpenTelemetry）
- 前端以 Zod 驗證 API 回應，或導入 openapi-typescript 產生型別
- CI（GitHub Actions）建置，並用 compose 跑 E2E 測試
